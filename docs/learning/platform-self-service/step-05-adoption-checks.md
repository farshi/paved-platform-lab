# Adoption Checks

## Concept

Adoption checks prove whether a team can use the platform path without platform-team hand-holding.

## Simple Meaning

Docs are not done because they exist. They are done when a team can complete the path.

## Why Teams Care

Teams need confidence that the platform will work when they are under delivery pressure.

## Runnable Example

Ask a learner to run:

```sh
make setup
make evidence
make tools-up
```

Then ask them to explain one policy block, one SLO panel, and one recovery action.

## What Can Go Wrong

If learners can only follow the demo with an expert narrating every step, the platform is not self-service yet.

## Platform Guardrail Or Best Practice

Use adoption checks as product feedback for the platform. Confusing docs, missing commands, and unclear dashboards are backlog items.

## Checkpoint

What would you change if a team can deploy but cannot explain rollback?
